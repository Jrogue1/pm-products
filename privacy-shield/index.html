How It Works

Three lines of code. Privacy for every message.

your-app.js
import
{ PrivacyShield }
from
'privacy-shield';

const
shield =
new
PrivacyShield({
  personalNames: ['John', 'Jane'],
  defaultMode: 'redact',
});

// Classify any message
const
result = shield.classify("My SSN is 123-45-6789");
// { score: 0.95, categories: ['pii'], recommendation: 'privacy' }


// Full routing decision
const
decision = shield.route(userMessage);
// { recommendedProvider: 'privacy', sanitized: { ... } }
1

Classify

Every message is scanned for PII, financial data, crypto addresses, medical records, legal content, and personal names. 40+ regex patterns with confidence scoring.

2

Sanitize

Sensitive content is redacted, masked, or tokenized before hitting external providers. Tokenization is reversible -- restore originals after processing.

3

Route

Get a routing decision: send to a privacy provider, standard provider, or block entirely. Your app decides what to do -- Privacy Shield tells you what's safe.

FAQ

What AI providers does Privacy Shield work with? ▼
Any of them. Privacy Shield is provider-agnostic middleware. It classifies and sanitizes messages before they reach any provider -- Claude, OpenAI, Venice, Ollama, or your own models. You configure which providers are "privacy" (zero retention) and which are "standard."
Does Privacy Shield store my messages? ▼
No. The audit log stores SHA-256 hashes of messages, sensitivity scores, and routing decisions -- never the actual message content. Token maps (for reversible tokenization) are held in memory only and never persisted to disk.
Is this a cloud service? ▼
No. Privacy Shield is a self-hosted Node.js module that runs on your machine. Your data never leaves your infrastructure unless you choose to send it to an AI provider -- and Privacy Shield helps you control exactly what gets sent.
What's the difference between one-time and subscription? ▼
Same software, different payment model. One-time gives you a perpetual license that never expires. Subscription is lower upfront cost with monthly billing. Both get the same features for their tier.
Can I add custom detection patterns? ▼
Yes, with the Pro tier. Pass custom regex patterns per category in your config. Industry-specific terms, internal identifiers, proprietary data formats -- whatever your business needs to detect.
What about HIPAA / SOC2 / GDPR compliance? ▼
Privacy Shield provides the technical controls: PII detection, data sanitization, audit trails, and routing enforcement. These are building blocks for compliance -- but compliance also requires organizational policies, training, and sometimes third-party certification that are outside the scope of any software tool.
